## 1 论文概览

### 1.1 一句话总结

FlexCAD 提出第一个利用大语言模型（LLM）实现"全层级可控"CAD 生成的统一模型：将 CAD 模型转换为简洁的结构化文本，再通过"层级感知掩码（hierarchy-aware masking）"策略对 LLM（Llama-3-8B + LoRA）进行微调，使其能够根据用户指定的任意构造层级（从粗粒度的草图-拉伸到细粒度的曲线）对 CAD 模型进行可控修改，且无需为不同层级训练不同模型。

### 1.2 研究背景与动机

CAD（计算机辅助设计）模型是 2D/3D 物体的数字表示，广泛应用于建筑、产品设计、制造等领域。常用 CAD 工具（如 SolidWorks、AutoCAD）普遍采用"草图-拉伸建模"（Sketch-and-Extrude Modeling, SEM）：先绘制 2D 草图，再拉伸成 3D 形状。相比 CSG（构造实体几何）、B-rep（边界表示）等表示方式，SEM 直接体现 CAD 的绘制过程，包含多个构造层级（草图-拉伸、拉伸、草图、面、环、曲线），便于编辑与复用。

近年来生成式 CAD 建模研究逐渐兴起：

- **DeepCAD** 专注于不可控生成（从随机采样向量生成 CAD 模型）；
- **SkexGen**、**Hnc-cad** 通过解耦码本（disentangled codebooks）提供一定程度的可控性，但存在明显局限：SkexGen 在 CAD 模型含多个草图时无法指定具体修改哪一个，也无法处理面、环等更细粒度的层级；Hnc-cad 缺乏对曲线层级拓扑和几何的控制。

总结来看，现有方法面临两大问题：（1）无法在所有构造层级上提供充分的可控性；（2）需要为不同类型的控制开发独立模型，效率低、实用性差。

作者受 LLM 近期进展的启发，认为 LLM 有潜力解决上述问题：第一，LLM 已证明能用单一统一模型处理多样化任务（涵盖自然语言及人体动作生成、晶体材料合成等领域特定微调任务）；第二，LLM 在预训练阶段可能已经通过学习 CAD 专用代码（如 JSCAD 代码）获得了 CAD 相关知识；第三，在 LLM 兴起之前，小型 Transformer 模型已在 2D 草图领域的不可控生成、图像到草图翻译任务中展现潜力。

## 2 核心方法：FlexCAD

### 2.1 总体框架

如原文[图1](#figure1)所示，FlexCAD 接收原始 CAD 模型以及用户想要修改的部分（高亮标注），用户可以在任意构造层级（草图-拉伸、拉伸、草图、面、环、曲线）指定该部分；FlexCAD 输出多个新 CAD 模型，仅改变被选定的部分。整体框架（原文[图2](#figure2)）分为两个核心步骤：

1. 将 CAD 模型转换为简洁、结构化的文本（Sec. 2.2）；
2. 引入层级感知掩码策略，对 LLM 进行微调以支持多种可控生成任务（Sec. 2.3）。

### 2.2 将 CAD 表示为结构化文本

#### （1）CAD 构造层级的定义

遵循 SEM 的传统定义（如原文[图3(a)](#figure3a)所示），CAD 模型存在多个构造层级：

- **曲线（Curve）**：基础层级，即直线 $l$、圆弧 $a$ 或圆 $c$，分别用 1、2、4 个点表示，每个点用 $x,y$ 坐标表示；
- **环（Loop）**：表示一条闭合路径，由单条曲线（如圆）或多条曲线（如"线-弧-线"）组成；
- **面（Face）**：2D 区域，由单个环或带有一个/多个内部环（作为孔洞）的外环构成；
- **草图（Sketch）**：由一个或多个共享同一拉伸命令的面组成；
- **拉伸（Extrusion）**：将草图从 2D 平面延伸到 3D 实体的命令；
- **草图-拉伸（Sketch-Extrusion）**：表示单个"草图+拉伸"形成的 3D 实体；
- **CAD 模型（CAD Model）**：由一个或多个草图-拉伸实体组成。

#### （2）结构化文本表示

如原文[图3(b)](#figure3b)所示，FlexCAD 将 CAD 模型表示为简洁的结构化文本：

- 曲线类型（直线/圆弧/圆）直接表示为文本 token；
- 点坐标（数值）转换为十进制整数后再转为文本 token——这与 SkexGen 采用的二值（one-hot）表示不同。例如将坐标离散化到 $64\times64$ 网格时，SkexGen 用 $([0,1,1,1,1,1],[0,1,1,1,1,1])$ 表示中心坐标，而 FlexCAD 直接表示为 $(31,31)$；
- 每个层级末尾添加特殊 token "`H_end`"（$H \in \{$curve, loop, face, sketch, extrusion$\}$）标记结束，区别于 SkexGen 使用 one-hot 向量作为结束标志；
- 细粒度层级的 token 依次拼接构成更粗粒度层级的表示（曲线→环→面→草图→拉伸→草图-拉伸→CAD 模型）。

相比 SkexGen，FlexCAD 的文本表示更简洁（整体 token 长度更短），且易于解析，便于后续层级感知掩码策略的实施。

### 2.3 基于层级感知掩码的 LLM 微调

#### （1）核心思路

在训练阶段，将 CAD 文本中某个"层级感知字段"（由一串 token 组成）替换为掩码 token，该字段可灵活设定以反映不同的构造层级；随后让 LLM 预测被掩码的字段（见原文[图2(a)](#figure2a)）。为此，作者为全部七个层级（CAD、草图-拉伸、拉伸、草图、面、环、曲线）设计了不同的提示模板（见原文[图4](#figure4)），这些模板在每个训练 epoch 中均匀采样，从而保证所有层级的生成任务都能在同一个统一模型中被学习。

#### （2）各层级的掩码设计

- **CAD 层级**：用 `[sketch-extrusion mask]` 掩码每一个内部草图-拉伸，此时除草图-拉伸数量外不保留原模型的任何信息，使推理时可以自由生成具有指定草图-拉伸数量的、复杂度不同的 CAD 模型；
- **草图-拉伸/草图/拉伸层级**：分别用 `[sketch-extrusion mask]`、`[sketch mask]`、`[extrusion mask]` 替换相应字段；
- **面/环层级**：若某面（环）单独构成一个草图（面），用 `[face mask]`（`[loop mask]`）掩码；若同一草图（面）下有多个面（环），用相应数量的掩码 token 同时掩码，使模型学会生成数量可变的面（环）；
- **曲线层级**：同一环内的所有曲线被掩码，且掩码 token 中标明曲线类型（line/arc/circle）。由于曲线是最基础的层级，草图的拓扑和几何控制均源于此——保持曲线类型和数量不变只能修改几何，改变类型或数量则可改变拓扑。

#### （3）统一训练（Unified Training）

每个 epoch，针对给定 CAD 文本，从上述七个层级中均匀采样一个提示模板，模板中的指令要求 LLM 自回归地预测被掩码字段，再用预测结果与模板答案之间的交叉熵（CE）损失反向传播更新模型。这种设计的优势有二：一是通过每轮随机选择已有模板，建立面向多层级的统一可控生成模型；二是可以方便地引入新模板以支持其他任务（如无条件生成）。

值得注意的是，FlexCAD 使用 **LoRA**（Low-Rank Adaptation）微调 LLM，只让少量参数可训练，同时保持大部分参数固定，兼顾大模型能力与收敛速度。此外，与 SkexGen、Hnc-cad 需要多阶段训练不同，FlexCAD 实现了**端到端训练**。

#### （4）基于用户自定义掩码的推理

推理时，CAD 模型先转换为 CAD 文本，用掩码 token 替换用户想要修改的部分，将该掩码文本输入微调后的 LLM 得到预测；用预测结果填充（infilling）掩码文本后，即可渲染出新的 CAD 模型。值得一提的是，用户在推理时不必严格遵循训练时的掩码模式——例如训练时同一面的多个环会被同时掩码，但推理时由于 LLM 强大的泛化能力，用户可以只掩码单个环以实现局部编辑（见原文[图2(b)](#figure2b)）。

## 3 实验设置与结果

### 3.1 数据集与实现细节

实验基于 **DeepCAD** 数据集（178,238 条草图-拉伸序列，按 90%-5%-5% 划分为训练/验证/测试集），并遵循 SkexGen 的做法移除重复和无效序列。基础模型选用 **Llama-3-8B**，用 LoRA 仅微调其 0.042% 的参数（约 340 万），LoRA rank/alpha 设为 8/32，4×A6000 GPU 训练，AdamW 优化器，batch size 32，余弦退火学习率 $5\times10^{-4}$，训练 30 个 epoch；推理阶段采样温度 $\tau=1.1$，Top-p$=0.9$。

### 3.2 评测指标

沿用 SkexGen、Hnc-cad 的评测体系：

- **Coverage (COV)**、**Minimum Matching Distance (MMD)**、**Jensen-Shannon Divergence (JSD)**：衡量生成多样性与质量（相对测试集而言）；
- **Novel**：生成模型中训练集未出现过的比例；
- **Unique**：生成集合中仅出现一次的模型比例；
- **Prediction Validity (PV)**：能被渲染为 3D 形状（而非仅 2D 草图或无效结果）的预测占比；
- **Realism**：人工评估的生成模型逼真度。

- **定性对比**（原文[图5](#figure5)）显示：FlexCAD 生成的模型结构更规整、更接近真实样例；SkexGen 无法指定具体修改哪个草图/拉伸（多个同名层级会被一并修改），Hnc-cad 即便从相同编码恢复也无法保持未掩码元素的完整性，而 FlexCAD 能确保仅修改被掩码的部分，其余部分保持不变；
- **人工评估（Realism）**：7 名众包工作者对比 950 对生成数据与训练数据图片，FlexCAD 的逼真度评分（39.6%/42.1%）在所有方法中最高；详细分布（原文[图8](#figure8)）显示 GPT-4o、SkexGen、Hnc-cad 的评分明显偏向"不够逼真"一端，而 FlexCAD 的分布接近对称，说明众包工作者难以区分生成模型与真实训练样本。

### 3.4 支持更多层级的可控生成任务

除草图、拉伸层级外，FlexCAD 还在 CAD、草图-拉伸、面、环、曲线层级实现了可控生成（原文[图6](#figure6)、附录[表5](#table5)），且不同层级间性能差异不大，证明了 FlexCAD 在全部层级上的有效性（详见原论文附录 A.3，CAD 层级 COV 67.2%，曲线层级 COV 59.1%，PV 均在 90% 以上）。

此外，FlexCAD 支持**迭代编辑（Iterative Editing）**：原文[图7](#figure7)展示了两个例子——(a) 从简单 CAD 模型出发，反复在新生成的草图-拉伸中编辑直至贴合用户需求；(b) 针对复杂 CAD 模型，在不同草图-拉伸内连续调整环-拉伸-面-曲线等多层级元素，逐步满足用户规格要求。

### 3.5 消融实验

#### （1）LLM 设置的影响（原论文 Table 2）

- 不使用预训练权重的 Transformer-4M 和 Llama-3-8B-From-Scratch 性能最低，说明 LLM 预训练阶段蕴含的知识对性能提升有重要贡献；
- Llama-3-8B-Full（全参数微调）性能与 FlexCAD（LoRA）相近，但需要 80 小时才能收敛，而 FlexCAD 仅需 20 小时，证明 LoRA 策略的高效性；
- Llama-3-8B-Instruct（指令微调版）性能略低于 FlexCAD；
- 随着模型规模显著增大，Llama-3-70B（仅微调 0.023% 参数，约 1630 万）取得最佳性能，但耗时更长、成本更高。

#### （2）层级感知掩码策略与统一训练的有效性（原论文 Table 3）

- **Random Masking**（随机掩码 15%-50% 连续 token 而非层级感知字段）性能最低，证明了"按层级掩码"的有效性；
- **w/o Hierarchy-specific Tokens**（使用通用 `[mask]` token 而非层级专属掩码 token）性能下降，说明层级专属 token 有助于 LLM 更准确判断当前生成的层级；
- **w/o Unified Training**（仅训练单一任务，如草图级可控生成）性能略有下降，说明同时训练多层级任务时存在的"跨任务知识"有助于整体性能提升。

### 3.6 无条件生成任务

FlexCAD 仅通过新增一个简洁提示模板（指令为"Below is a description of a CAD sequence:"，答案为完整 CAD 文本）即可实现无条件 CAD 生成（原论文 Table 7、附录[图17](#figure17)），生成质量整体优于 SkexGen 和 Hnc-cad（COV 89.2%、PV 90.5%），但 JSD 表现较差（1.53），不过调整采样温度/Top-p 将 PV 维持在约 80% 时，JSD 可显著改善至 0.78。

### 3.7 关键超参数敏感性分析

针对采样温度 $\tau$ 与 Top-p（原论文 Table 6）的分析显示：$\tau$ 或 Top-p 越大，前五项指标（COV、MMD、JSD、Novel、Unique）表现越好，但 PV（预测有效性）随之下降——更高的随机性带来更多样化的预测，但有效性降低。作者最终选择能保证 PV 维持在 90% 以上的取值（$\tau=1.1$，Top-p$=0.9$）作为折中方案。

## 4 局限性与失败案例（作者自述）

### 4.1 推理效率

在单张 A6000 GPU（batch size = 1）上测得拉伸级生成的平均推理时间：FlexCAD（Llama-3-8B）为 0.56 秒，略高于 SkexGen（0.15 秒）和 Hnc-cad（0.38 秒）；Llama-3-70B 版本（4×A100）平均推理时间接近 3 秒。作者认为，尽管 LLM 普遍缺乏效率，但可控 CAD 生成任务对实时性要求不高，因此该推理时间的小幅增加是可以接受的。

### 4.2 幻觉与失败案例

如原文[图9](#figure9)所示，在曲线层级可控生成中，当用 5、6、7、8 条直线替换原始环时，FlexCAD 倾向于生成"看似合理"但实际呈现不规则多边形的 CAD 文本；相比之下，限定为 4 条直线时生成结果更规整。作者分析训练集中由 4/5/6/7/8 条线构成的环的比例约为 77.9%-6.2%-9.5%-1.7%-4.7%，认为幻觉的根本原因可能源于**数据不平衡**，可通过收集更多数据来缓解。

### 4.3 未探索方向

论文仅探索了基于"层级掩码"的可控 CAD 生成，尚未涉及**纯文本描述驱动的 CAD 生成**（如输入"Create a desk."生成对应模型）。作者将这一方向（构建面向文本到 CAD 生成任务的多模态 LLM）列为未来工作。

## 5 内容要素总结

### 5.1 解决何问题

如何用一个统一模型实现 CAD 模型在**所有构造层级**（草图-拉伸、拉伸、草图、面、环、曲线）上的可控生成，避免现有方法（SkexGen、Hnc-cad）存在的"可控性受限于特定层级"和"不同控制类型需要不同模型"两大问题。

### 5.2 大致采取何方法

1. 将 CAD 模型的草图-拉伸建模过程抽象为多层级结构，并设计简洁的结构化文本表示，把曲线类型、点坐标、拉伸参数等都转换为文本 token，并用特殊结束符标记各层级边界；
2. 提出层级感知掩码策略：训练时按层级随机选取一个字段用掩码 token 替换，配合七套不同层级的提示模板，让 LLM 学习预测被掩码的字段；
3. 利用 LoRA 对 Llama-3-8B 进行端到端微调，实现统一、高效的训练；
4. 推理时，用户在 CAD 文本中标记想修改的任意层级部分为掩码，模型生成预测后填充并渲染为新 CAD 模型，支持灵活的局部编辑和迭代编辑。

### 5.3 优势与不足

**优势：**

- 首次将 LLM 引入可控 CAD 生成领域，用单一统一模型同时支持 CAD/草图-拉伸/草图/拉伸/面/环/曲线七个层级的可控生成，避免了为不同控制类型分别建模；
- 结构化文本表示简洁（比 SkexGen 的 one-hot 编码更短），便于 LLM 理解和处理，且实现端到端训练（无需像 SkexGen、Hnc-cad 那样多阶段训练）；
- 在草图级和拉伸级任务上全面超越 SkexGen、Hnc-cad、GPT-4o（few-shot），尤其在预测有效性（PV）和人工评估的逼真度（Realism）上提升显著；
- 使用 LoRA 微调大幅降低训练成本（收敛时间从全参数微调的 80 小时降至 20 小时），同时保持与全参数微调相近的性能。

**不足：**

- 推理效率低于专用模型 SkexGen 和 Hnc-cad（0.56s vs 0.15s/0.38s），更大规模模型（Llama-3-70B）推理耗时进一步增加（约 3 秒），LLM 固有的推理效率问题尚未解决；
- 存在因训练数据分布不平衡导致的幻觉问题：在曲线层级生成不常见线段数量（如 5、6、7、8 条线）的环时，容易生成不规则多边形等不合理几何结构；
- 目前仅支持"结构化文本/层级掩码"驱动的可控生成，尚不支持自然语言描述（如"创建一张桌子"）直接驱动的文本到 CAD 生成，应用场景的灵活性有限；
- 无条件生成任务中 JSD 指标表现欠佳（除非牺牲部分 PV），表明模型在不施加层级约束时生成分布的整体匹配度仍有提升空间。

### 5.4 创新点

- **结构化文本表示**：用直观的十进制整数 token 替代 SkexGen 的 one-hot 二值编码表示坐标，并用层级专属结束符标记边界，使 CAD 模型表示更简洁、更易被 LLM 理解和处理；
- **层级感知掩码策略**：首次提出按 CAD 构造层级（而非随机连续片段）进行掩码，并为每个层级设计专属掩码 token 与提示模板，是实现"全层级可控、单一统一模型"的核心机制；
- **统一训练范式**：通过在每个训练 epoch 均匀采样七类层级模板，让单一 LLM 同时学习多层级生成任务，相比单任务训练能获得额外的跨任务知识收益；
- **基于 LoRA 的高效微调**：相比全参数微调，大幅降低训练成本而不损失生成质量，且支持模型规模灵活扩展（Llama-3-8B → 70B）。

### 5.5 个人感悟

FlexCAD 最值得借鉴的思路在于把"可控性"问题转化为"掩码预测"问题——这与 NLP 领域的掩码语言建模（如 BERT 的 MLM、T5 的 span corruption）一脉相承，只是将掩码对象从文本片段替换为具有明确语义层级的 CAD 构造单元。这种"领域结构 + 通用掩码预测范式"的结合方式简单却高效，使得论文标题中的"Unified and Versatile"真正落地：用同一个模型权重，仅通过改变提示模板/掩码方式，就能服务于从粗粒度（草图-拉伸数量控制）到细粒度（单条曲线的拓扑/几何调整）的全谱系可控生成需求，这种设计思路对其他具有层级化结构的领域生成任务（如分子结构生成、电路设计、3D 场景布局）具有较强的迁移参考价值。

但也应注意到，FlexCAD 的可控性本质上仍局限于"用户能够指出 CAD 文本中的哪个字段需要修改"——这要求用户对 CAD 模型已有的层级结构有一定理解，与"自然语言直接描述设计意图"（如"把这个零件改得更扁一些"）之间仍有较大差距，这也正是作者在未来工作中提到的方向。此外，论文披露的"曲线数量分布不均导致幻觉"问题，揭示了 LLM 在处理结构化、组合性强的生成任务时仍然容易受训练数据分布长尾效应的影响，这提示在将 LLM 应用于工程/设计类结构化生成任务时，数据均衡性和稀有结构的覆盖度可能是比单纯扩大模型规模更值得优先解决的问题。
