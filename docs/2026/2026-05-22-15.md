## 1 绪论与研究背景
### 1.1 研究背景与动机
计算机辅助设计（CAD）模型的核心价值在于其构建历史（Construction History）：一种能够编码设计意图的参数化配方 。这种结构允许工程师修改尺寸、重放历史并集成约束，而这些能力在纯几何表示（如网格或点云）中会完全丢失 。

然而，在生成式 CAD 领域存在着严重的数据匮乏问题 [cite: 32]。虽然现有的百万级 3D 数据集（如 ABC、Objaverse）提供了大量的模型，但它们仅包含边界表示（B-Reps）或网格，剥离了底层的参数化历史 。少数包含构建序列的数据集（如 DeepCAD、Fusion 360 Gallery）规模较小，且基本局限于简单的“草图-拉伸”工作流，缺少倒角、圆角和布尔操作等真实工业设计中丰富的操作词汇 。而像 CAD-Recode 这样的程序化合成方法，生成的代码往往缺乏语义深度，仅是坐标链的转译，缺乏人类工程师编写的逻辑结构，难以被人类阅读和编辑 。

### 1.2 核心研究问题
针对上述问题，本研究提出了一个核心假设：大语言模型（LLM）已经从海量文本中吸收了关于物体结构和制造工艺的丰富知识。例如它们知道支架需要安装孔，轴需要键槽，即使它们不原生支持 CAD 内核的精准语法 。那么，能否利用计算资源，通过智能体合成（Agentic Synthesis）将 LLM 内部潜在的机械设计知识转化为可执行、可读性强的 CAD 程序，从而在无需真实构建历史数据的前提下，实现百万级高质量数据集的无中生有？

## 2 Zero-to-CAD 管道架构
### 2.1 智能体代码合成与验证环境
本研究将 CAD 脚本合成任务建模为一个智能体搜索与自纠错问题。该系统不再要求模型“单发”（One-shot）生成完美的程序，而是将 LLM 置于一个带有反馈驱动的闭环 CAD 环境中。

智能体在每轮交互中能够：编写代码、在隔离环境内执行、观察捕获的错误与几何拓扑指标、查阅 API 文档并自行修正代码。整个基础设施由以下四个协同工作的组件构成：

* LLM 推理服务：部署在基于 vLLM 的 Ray 集群上，利用 KV 缓存实现高效的多轮推理，并通过函数调用支持水平扩展。
* 协调节点：负责管理独立的智能体 rollout 过程，处理负载均衡、容错和产物串流。
* 具备工具的 Workers：为智能体提供三个核心工具：
  1. execute_and_validate：在隔离子进程中执行 CadQuery 代码，进行多阶段几何验证，并返回包含错误信息、拓扑指标和导出状态的结构化反馈。
  2. lookup_documentation：基于 TF-IDF 对 CadQuery API 文档进行轻量级检索。
  3. grep_documentation：基于正则表达式进行精确的语法匹配，防止 TF-IDF 检索结果过于宽泛 。
* 存储后端：将通过所有验证的代码、STL 网格、STEP 文件以及元数据实时流式传输至云存储。

### 2.2 两阶段生成协议
为了在维持几何有效性的同时促进控制类别下的多样性，管道引入了两阶段生成协议：

#### （1）阶段一：目录生成（Catalog Generation）
LLM 扮演“机械文献管理员”的角色，在大上下文窗口内以大批量（通常为 200 个）的形式生成不含具体尺寸的简短工程零件描述，利用上下文避免批次内的重复。最终生成简洁且不含具体尺寸的规格说明（如：“一个带有两个通孔的安装支架”），去重后索引供后续使用。

#### （2）阶段二：基于描述的代码生成
利用阶段一生成的描述驱动代码合成。零件 Worker 接收描述、包含 19 条设计原则的系统提示词以及（可选的）高级参考代码片段。模型被明确要求将参考片段作为技术参考进行迁移学习，严禁直接复制参数，必须针对新描述生成独创几何。

### 2.3 多阶段验证框架与去退化机制
多阶段验证是确保百万级数据可用性的技术护城河。仅通过编译的代码无法保证几何实体的正确性，因此框架执行以下三重校验：
* 代码执行验证：验证基本语法和运行时行为，捕获完备的堆栈轨迹供模型反馈。
* 几何有效性验证：包括拓扑有效性（无自交或退化面）、连通性检查（必须为单一连通实体，杜绝不完整的布尔并集导致的多断体）以及正体积校验。
* 导出验证：强制进行 STL 和 STEP 格式导出测试，用以揭示无法被拓扑检查直接捕获的精细几何错误（如无效的面朝向）。

:::info[去退化核心策略]
在多轮修复循环中，系统提示词严格禁止模型通过“简化代码”或“删除特征”的方式来消解报错。智能体被强力约束必须通过查阅正确语法来维持预期的几何复杂度，从而完全阻止了系统向平庸的低面数简单几何退化的趋势。
:::

### 3.3 可读性与可编辑性对比
在代码表征层面上，Zero-to-CAD 生成的是具有高级逻辑抽象的代码。CAD-Recode 的转译代码由没有参数结构的坐标链组成（例如连续使用 .segment 绘图并直接挤压数值），这使得人类工程师极难手动编辑 。相比之下，Zero-to-CAD 程序使用显式的命名参数（如板厚、圆角半径）和符合人类直觉的逻辑构建顺序，使设计意图更加明确，修改也更加简单直接。

## 4 自举（Bootstrapping）多视图图像重构实验
为验证该数据集在缺乏真实工业构建历史数据的现状下能否有效赋能下游模型，研究设计了多视图图像到 CAD 序列（Image-to-Sequence）的重构自举实验。

### 4.1 实验设置
* 任务定义：输入给模型某个零件的 8 张分辨率为 256x256 的多视图渲染图（4张前视，4张后视），要求模型在单次前向传播中，直接预测出能够复现该几何实体的可执行、参数化 CadQuery 代码。
* 骨干模型：完全微调包含视觉编码器、语言解码器和 MLP 适配器的 Qwen3-VL-2B-Instruct 基础模型。
* 基准线（Baselines）：包含未经过微调的原始 Qwen 视觉模型（建立零样本基线），以及在 Medium 和 High 两种推理级位上的闭源商用前沿模型 GPT-5.2。
* 评估指标：三维空间体素交并比（Voxel IoU，64^3 分辨率，在 45° 步长旋转对齐下取最大值）以及倒角距离（Chamfer Distance, CD。

## 5 论文核心要素总结与深入评析
### 5.1 解决的瓶颈问题
成功化解了三维生成式 CAD 领域长期面临的“高价值参数化构建序列数据极端匮乏，而专有设计数据受制于格式壁垒与知识产权无法公开”的死结。

### 5.2 大致采取的方法
将大语言模型封装在包含多阶段拓扑-体积连通性检查及 API 文档图谱检索的闭合交互回路中。通过由文本描述驱动的操作选择机制和强力去退化策略，让 LLM 的通用设计先验在严格的代码执行边界下进行高效的空间发散和自我修正，实现了百万规模、具备工业可编辑性的 CadQuery 脚本池资产的纯合成生产。

### 5.3 创新点
1. 解耦式的两阶段设计哲学：首创通过阶段一的“机械文献管理员角色”在大上下文窗口中统一规划几何的多样化骨架，在解耦“建造什么”和“如何建造”的同时，天然避免了模式塌陷。
2. 强制性的几何自纠错去退化协议：在多轮迭代中注入底层物理及拓扑断开性强校验，并从机制上剥夺模型通过精简特征妥协通过测试的权利，将算力真正转化为有效的高熵复杂数据。
3. 成功打通“纯合成数据至真实世界”的自举闭环：通过微调极轻量级（2B）的端到端视觉模型，在没有物理世界真实历史轨迹喂养的严苛环境下，实现了针对真实人类手稿级模型高保真度的多视角逆向工程。

### 5.4 优势与不足
* 显着优势：突破了传统 CAD 生成方法过度依赖单一的“草图-拉伸”局限，首次将布尔、放样、扫掠、圆角等高级复杂建模特征大规模推向百万量级数据集；代码层级保留了显式的变量解耦和富有可读性的参数，完美兼容现有工业流水线的二次工程调整。
* 关键不足：
  1. 缺乏对制造可行性（DFM）规则的深层绑定：当前验证框架虽然保证了解析几何和拓扑空间的合法性，但并没有实质引入工艺依赖性约束（如 CNC 加工的刀具干涉、压铸模具的拔模斜度细节等）。
  2. LLM 固有的全局三维空间逻辑盲区：从失效模式来看，由于 LLM 底层对 3D 空间关系的推演顺应纯粹的文本逻辑，对于薄壁干涉、局部相交以及长程序带来的全局尺度漂移仍存在一定的长尾误判。
